# Analysis of Sorting Algorithms and Convergence in PreferenceRank

This document summarizes the analysis performed to optimize the pair generation and scoring system in PreferenceRank.

## 1. Sorting Algorithm Comparison (N=100)

We compared several sorting algorithms to determine which provides the best balance between the number of required battles (comparisons) and the accuracy of the resulting ranking.

### Benchmarking Methodology
- **N Value:** 100
- **Trials:** 50 per algorithm
- **Metric 1: Avg Battles:** The average number of comparisons generated by the algorithm.
- **Metric 2: Avg Kendall Tau:** A measure of rank correlation between the true hidden strengths and the estimated scores (1.0 is perfect correlation).

### Results (N=100), Sorted by Kendall Tau
| Algorithm | Avg Battles | Avg Kendall Tau | Pareto Status |
| :--- | :--- | :--- | :--- |
| Heapsort | 146.64 | 0.4888 | Pareto-optimal |
| Insertion Sort | 2585.82 | 0.8060 | Dominated |
| Quicksort | 640.98 | 0.8368 | Dominated |
| Binary Insertion | 530.06 | 0.8872 | Dominated |
| **Ford-Johnson** | **526.62** | **0.8897** | **Pareto-optimal** |
| Merge Sort | 543.24 | 0.9035 | Pareto-optimal |
| Selection Sort | 4950.00 | 0.9330 | Dominated |
| **Shellsort** | **728.70** | **0.9424** | **Knee Point** |
| Bubble Sort | 4873.08 | 0.9732 | Dominated |
| Cocktail Shaker | 3897.50 | 0.9785 | Dominated |
| Comb Sort | 1238.62 | 0.9904 | Pareto-optimal |
| Full Rank | 4950.00 | 1.0000 | Pareto-optimal |

### Pareto Frontier & Knee Point Analysis
The **Pareto Frontier** identifies algorithms where no other algorithm is both better at minimizing battles and better at maximizing accuracy.

- **Ford-Johnson** is Pareto-optimal and provides a very high accuracy-to-battle ratio. It is theoretically optimal for small N and remains extremely efficient at N=100.
- **Shellsort** is identified as the **mathematical knee point** for N=100. It offers a significant jump in accuracy (0.94) for a moderate increase in battles compared to Ford-Johnson.
- **Full Rank** is the absolute baseline for accuracy but requires a prohibitive number of battles (4950).

**Conclusion:** While Shellsort is the mathematical knee point at N=100, **Ford-Johnson** is chosen for "Quick Rank" because it maintains high accuracy (~0.89) with the absolute minimum number of comparisons needed to establish a total ordering, which is crucial for reducing user fatigue in an interactive setting.

## 2. Bradley-Terry Convergence Analysis

We analyzed the Minorization-Maximization (MM) algorithm's convergence to find a "knee point" where further iterations do not significantly improve score accuracy.

### Benchmarking Methodology
- **Thresholds:** 1e-1 to 1e-12
- **Ref Score:** Calculated at 1e-15 threshold.
- **Metric 1: Iterations Saved %:** Reduction in iterations compared to a 1e-12 baseline.
- **Metric 2: Max Error:** Maximum absolute difference in scores compared to the reference.

### Results (N=100, Density=2)
| Threshold | Iterations | Iter Saved % | Max Error |
| :--- | :--- | :--- | :--- |
| 1e-1 | 8 | 97.0% | 1.27e+2 |
| 1e-3 | 50 | 81.5% | 1.57e+0 |
| 1e-5 | 96 | 64.4% | 1.56e-2 |
| **1e-7** | **144** | **46.7%** | **1.79e-4** |
| 1e-12 | 270 | 0.0% | 5.35e-9 |

### The Knee Point
A threshold of **1e-7** is identified as the **knee point** for convergence.
- It consistently saves **~45-47%** of computational iterations.
- The maximum score error is approximately **0.0002**, which is several orders of magnitude below the precision of the integer scores shown to the user.

**Conclusion:** Setting the convergence threshold to `1e-7` provides a significant performance boost (~2x speedup in scoring) with no perceptible loss in accuracy.
